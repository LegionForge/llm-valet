## Test 6 — Context window selector: validation

**What we're testing:** The backend rejects invalid `num_ctx` values with a 422 response.

```bash
# Should succeed — valid num_ctx
curl -s -X POST http://localhost:8764/load \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <api-key>" \
  -d '{"model": "qwen3.5:0.8b", "num_ctx": 4096}' | python3 -m json.tool

# Should fail — below minimum (< 512)
curl -s -X POST http://localhost:8764/load \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <api-key>" \
  -d '{"model": "qwen3.5:0.8b", "num_ctx": 256}' | python3 -m json.tool

# Should fail — string instead of int
curl -s -X POST http://localhost:8764/load \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <api-key>" \
  -d '{"model": "qwen3.5:0.8b", "num_ctx": "big"}' | python3 -m json.tool
```

| # | Check | Pass? | Notes |
|---|---|---|---|
| 6a | Valid `num_ctx: 4096` → `{"ok": true, "num_ctx": 4096}` | PASS | Confirmed |
| 6b | `num_ctx: 256` → HTTP 422, `"num_ctx must be an integer >= 512"` | PASS | `{"detail": "num_ctx must be an integer >= 512"}` |
| 6c | `num_ctx: "big"` → HTTP 422 or 400 | PASS | Same 422 detail — coerced through int() check, correctly rejected |

## Known issues / deferred

| Issue | Status | Notes |
|-------|--------|-------|
| GPU bar not shown on Apple Silicon (VRAM null) | By design | M4 unified memory not exposed via discrete VRAM API |
| svcmgr/linux.py + windows.py not tested here | Deferred | macOS test target only; Linux/Windows paths exercised by unit tests |
| Context selector hidden when model context_length unknown | By design | Row hidden rather than showing meaningless options |
| RESUME reverts to Ollama default ctx (4K) after PAUSE | Bug — fix in v0.5 | resume() calls keep_alive:-1 without num_ctx. Valet does not persist the ctx override across pause/resume cycles. Fix: store active num_ctx in ProviderStatus, pass through resume(). |
| RAM bar underreports load for swap-heavy models on Apple Silicon | By design / known | psutil reports physical RAM pressure only. Models > physical RAM (e.g. mistral-nemo:12b at 128K ctx = 25 GB on 16 GB machine) spill to macOS swap. RAM% shows ~71% while model is 25 GB. `memory_used_mb` from Ollama is authoritative for actual model size. |
| Model dropdown shows `— select model —` after hard-refresh | Minor — fix in v0.5 | renderProvider() fires before loadModels() adds options on page load; can't pre-select loaded model. Resolves on next poll tick (≤5s). |
| Integration tests (api.py, ollama.py, watchdog.py) | Deferred to v0.5 | 0% covered — unit tests only |
