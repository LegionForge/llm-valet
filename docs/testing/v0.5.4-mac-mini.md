## Test E4 — Config File Abuse (Manual)

**What we're testing:** Service handles corrupt or missing config gracefully.

```bash
# Corrupt config — backup first
cp ~/.llm-valet/config.yaml ~/.llm-valet/config.yaml.bak
echo "host: [invalid yaml{{{{" > ~/.llm-valet/config.yaml
launchctl kickstart -k gui/$(id -u)/com.legionforge.llm-valet
# wait 5s, then check if service is still up or errored
curl http://localhost:8765/status

# Restore
cp ~/.llm-valet/config.yaml.bak ~/.llm-valet/config.yaml
launchctl kickstart -k gui/$(id -u)/com.legionforge.llm-valet
```

| # | Check | Pass? | Notes |
|---|---|---|---|
| E4a | Service restart with corrupt config.yaml — does not panic, logs a clear error | | |

| E4c | Service recovers cleanly after config restored and restarted | | |
| E4d | Delete `~/.llm-valet/config.yaml` while service running — service continues on in-memory config | | |
