## Test E8 — Security Fault Injection (Manual)

**What we're testing:** Active probing of the security mitigations from the threat model (T1–T8 in CLAUDE.md). All tests run from the LAN IP with correct API key unless noted.

```bash
# Set these before running
HOST=10.0.100.203
PORT=8765
KEY=<api-key>
```

### E8a — Injection in model name (T4)

```bash
# Shell metacharacters in model name — should be rejected 422
curl -s -X POST "http://$HOST:$PORT/load" \
  -H "X-API-Key: $KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen3.5:0.8b; rm -rf /"}'

# Command substitution attempt
curl -s -X POST "http://$HOST:$PORT/load" \
  -H "X-API-Key: $KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "$(whoami)"}'
```

### E8b — Oversized request body (T1)

```bash
# 1 MB body to PUT /config — should be rejected, not hang or OOM
python3 -c "import json; print(json.dumps({'ram_pause_pct': 85.0, 'padding': 'A'*1048576}))" | \
  curl -s -X PUT "http://$HOST:$PORT/config" \
  -H "X-API-Key: $KEY" \
  -H "Content-Type: application/json" \
  -d @-
```

### E8c — Host header injection (T2 — DNS rebinding)

```bash
# Spoofed Host header — TrustedHostMiddleware should return 400
curl -s -o /dev/null -w "%{http_code}" \
  -H "Host: evil.com" \
  "http://$HOST:$PORT/status"
```

### E8d — CORS origin probe (T3)

```bash
# Cross-origin preflight — Origin should NOT be reflected with allow-all
curl -s -I -X OPTIONS "http://$HOST:$PORT/status" \
  -H "Origin: http://evil.com" \
  -H "Access-Control-Request-Method: GET"
# Check: Access-Control-Allow-Origin must NOT be * or http://evil.com
```

### E8e — SSRF via ollama_url (T6)

```bash
# Try to redirect provider URL to an internal service
curl -s -X PUT "http://$HOST:$PORT/config" \
  -H "X-API-Key: $KEY" \
  -H "Content-Type: application/json" \
  -d '{"ollama_url": "http://169.254.169.254/latest/meta-data/"}'
# Should be rejected — only localhost/RFC1918 Ollama URLs allowed
```

### E8f — Credential stuffing (T1)

```bash
# 10 rapid wrong-key requests — should all 401, no lockout panic, no 500
for i in $(seq 1 10); do
  curl -s -o /dev/null -w "%{http_code}\n" \
    -H "X-API-Key: wrongkey$i" \
    "http://$HOST:$PORT/status"
done
```

| # | Check | Pass? | Notes |
|---|---|---|---|
| E8a-1 | Model name with shell metacharacters `;` — rejected, not executed | ✅ | Fixed in v0.5.4 — `_validate_model_name()` rejects names not matching `^[a-zA-Z0-9:._-]+$`; returns 422 |
| E8a-2 | Model name with `$(whoami)` — rejected, not executed | ✅ | Same fix; safe regardless due to `shell=False` throughout, but input validated at boundary |
| E8b | 1 MB body to PUT /config — rejected cleanly, no hang or OOM | ✅ | Fixed in v0.5.4 — 64 KB body size limit middleware; returns 413 |
| E8c | Spoofed `Host: evil.com` — returns 400, request blocked | ✅ | TrustedHostMiddleware working correctly — pre-existing |
| E8d | `Origin: http://evil.com` preflight — not reflected in CORS headers | ✅ | CORS allow_origins=[] — evil.com not reflected — pre-existing |
| E8e | `ollama_url` set to AWS metadata endpoint — rejected with validation error | ✅ | False positive — `ollama_url` is not a `ResourceThresholds` field; PUT /config silently ignores it (unknown key path). `_validate_provider_url()` enforces RFC1918-only at config load time |
| E8f | 10× wrong API key — all 401, no 500, service remains responsive | ✅ | All 401, no degradation |
