# llm-valet v0.6.0 — End-to-End Validation: Mac Mini
**Date:** TBD
**Type:** End-to-end hardware validation (requires Mac Mini + live Ollama)
**Tester:** JP Cruz
**Target hardware:** Mac Mini M4 (16 GB unified memory)
**Branch / tag:** `v0.6.0`

This is the gate test for v0.6.0 → v1.0 promotion. Two scenarios must both pass:
- **E — Upgrade path** (existing install → v0.6.0)
- **C — Clean install** (fresh machine / wiped state → v0.6.0 from PyPI)

Both scenarios start from a machine with Ollama already installed via Homebrew.

## Known gotchas

**macOS launchd context** — The Ollama launchd agent may lack user home directory access on macOS 15 (Sequoia) if it was installed before the system privacy settings were configured. If `ollama list` returns empty but models exist in `~/.ollama/models/manifests/`, restart Ollama from a terminal session:
```bash
brew services stop ollama
ollama serve &
```

Then re-run the integration tests (`E5`).

**Integration tests require terminal-context Ollama** — `pytest -m integration` must be run while Ollama is running in a terminal session (not as a launchd service) due to the macOS 15 privacy restriction documented above. This affects test execution only; the installed service is unaffected for normal use.
