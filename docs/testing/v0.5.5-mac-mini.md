## Test U2 — Watchdog FSM Coverage

**What we're testing:** State machine transition paths that were not exercised by the existing test suite. See [docs/watchdog-fsm.md](../watchdog-fsm.md) for the full state diagram.

Previously untested transitions:
- RESUMING → PAUSED (resume call fails)
- PAUSED no-op when game is still running (should_pause=True while already PAUSED)
- PAUSED stays PAUSED in hysteresis zone (RAM above resume% but below pause%)
- GPU VRAM above threshold triggers pause + sets `_pause_trigger = "gpu"`

| # | Check | Pass? | Notes |
|---|-------|-------|-------|
| U2a | `provider.resume()` returns False → state stays PAUSED | ✅ | |
| U2b | PAUSED + game still running → no pause() or resume() call (pure no-op tick) | ✅ | |
| U2c | RAM in hysteresis zone (70%, resume=60%, pause=85%) → stays PAUSED | ✅ | |
| U2d | GPU VRAM at 90% (threshold 85%) → PAUSED, `_pause_trigger == "gpu"` | ✅ | |
| U2e | GPU VRAM at 70% (below threshold) → stays RUNNING | ✅ | |
