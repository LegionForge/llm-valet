## Test 2 — Force PAUSE against an active inference request

**What we're testing:** The core scenario for issue #5. A long-running inference request holds the model in memory. Normal PAUSE is sent but the model stays loaded (Ollama queues the eviction). FORCE PAUSE kills the runner process directly, evicting the model immediately.

**How it works:** Two terminal windows needed — one for the blocking inference, one for control.

### Step 1 — Load a model and start a long inference in the background

```bash
# Terminal A: start a long streaming inference (runs for 30-60s)
curl -s -X POST http://localhost:8764/load \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <api-key>" \
  -d '{"model": "qwen3.5:0.8b", "num_ctx": 4096}'

# Confirm model is loaded
curl -s http://localhost:11434/api/ps | python3 -m json.tool | grep '"name"'

# Start a long streaming inference — leave this running in Terminal A
curl -s -X POST http://localhost:11434/api/generate \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen3.5:0.8b", "prompt": "Write a detailed 2000-word essay about the history of computing, covering all major milestones from Babbage to modern GPUs.", "stream": true}' \
  > /dev/null &
INFERENCE_PID=$!
echo "Inference running as PID $INFERENCE_PID"
```

### Step 2 — Send PAUSE while inference is running

```bash
# Terminal B: send PAUSE while inference is still in progress
curl -s -X POST http://localhost:8764/pause \
  -H "X-API-Key: <api-key>" | python3 -m json.tool
```

| # | Check | Pass? | Notes |
|---|---|---|---|
| 2a | `/pause` returns `{"ok": true}` | | |
| 2b | `/status` shows watchdog state `"paused"` | | |
| 2c | BUT: `/api/ps` still shows the model loaded (inference blocking eviction) | | |

```bash
# Confirm model is still loaded (Ollama hasn't evicted yet — inference is running)
curl -s http://localhost:11434/api/ps | python3 -m json.tool | grep '"name"'
```

### Step 3 — Send FORCE PAUSE

```bash
# Still in Terminal B — send force pause
curl -s -X POST http://localhost:8764/pause/force \
  -H "X-API-Key: <api-key>" | python3 -m json.tool
```

| # | Check | Pass? | Notes |
|---|---|---|---|
| 2d | `/pause/force` returns `{"ok": true, "action": "force_pause"}` | | |
| 2e | `/api/ps` is now **empty** — model evicted immediately | | |
| 2f | Background inference (Terminal A) was interrupted — `curl` process exits/errors | | |
| 2g | `/status` watchdog state remains `"paused"` | | |
| 2h | No errors in `~/.llm-valet/valet.stderr.log` beyond expected runner kill | | |

```bash

curl -s http://localhost:11434/api/ps | python3 -m json.tool
```

## Test 3 — Force PAUSE with no active inference (fallback path)

```bash

curl -s http://localhost:11434/api/ps | python3 -m json.tool | grep '"name"'
```

## Test 4 — ctx preserved through force_pause → RESUME

**What we're testing:** After a force pause, `_last_loaded_ctx` was captured before the runner kill. RESUME should restore the same context window.

## Known constraints

| Item | Notes |
|------|-------|
| Runner process detection | On Apple Silicon, Ollama spawns `ollama_llama_runner` as a subprocess of the app bundle. If the process name differs, T2 will show the fallback path (log: "no runner processes found") — document the actual process name in Notes column. |
| Inference blocking behavior | Ollama may queue the `keep_alive: 0` eviction or return immediately depending on version. If PAUSE already evicts before 10s, T5b may not be testable in a single attempt — try a longer inference prompt. |
