## Session 2026-04-05 — Remote testing findings

**Testing approach:** Tests 4b–11 run remotely from Windows dev machine to Mac Mini via X-API-Key auth over LAN. Tests 1–3 and 13 done on-device.

### Key findings

**Finding 1 — Watchdog oscillation on 16GB M4 with large models**  
`qwen3.5:latest` (8GB) pushes RAM to ~88% when loaded. Default `ram_pause_pct: 85%` is below this, causing the watchdog to pause immediately after every auto-resume. This creates an infinite pause/resume loop.  
**Recommended config for this machine:** `ram_pause_pct: 95.0` (model stays loaded at ~88%).  
**General principle:** `ram_pause_pct` must be set above the loaded-model RAM level to avoid oscillation.

**Finding 2 — /stop fails on macOS with Homebrew LaunchAgent (FIXED)**  
`OllamaProvider.stop()` was using psutil SIGTERM directly. On macOS, Ollama's Homebrew LaunchAgent causes launchd to respawn the process immediately after SIGTERM. stop() would wait 30s, fail, and return `ok:false`.  
**Fix applied 2026-04-05:** `stop()` now calls `svcmgr.macos.stop_service()` (launchctl bootout) first, then falls back to psutil SIGKILL only if needed. Similarly, `start()` now calls `svcmgr.macos.start_service()`.  
**Needs re-test on Mac Mini** (Test 11 re-run).

**Finding 3 — Model load time from external drive**  
Models on SATA3 over USB-A take longer to load than the provider's 15s httpx timeout. `resume()` returns `ok:true` (HTTP 200 received from Ollama stream start) but the model may not be in `/api/ps` immediately. Check `model_loaded` status 5–10s after /resume, not immediately.

**Finding 4 — Remote testing is viable**  
All functional tests (4b–10) run cleanly over the LAN with `X-API-Key` auth. Test infrastructure: curl from Windows to Mac Mini via IP. Tests 8, 12, 13, and the MANUAL items above require on-device access.
