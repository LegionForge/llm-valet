## Test 6 — STOP button actually stops Ollama (svcmgr fix)

**What we're testing:** The STOP button sends `bootout` to launchd via the correct plist (`homebrew.mxcl.ollama.plist`), preventing KeepAlive respawn. This was broken in v0.1.

```bash
KEY="e99fde85d8c6ed3c9e3380b58d47b50334587b30cb2113abb9120e406eab702a"

# Trigger STOP
curl -s -m 5 -X POST -H "X-API-Key: $KEY" http://localhost:8764/stop

# STOP returns immediately (non-blocking). Poll for Ollama going down.
for i in $(seq 1 15); do
  sleep 2
  status=$(curl -s --connect-timeout 2 http://localhost:11434/api/tags 2>/dev/null && echo "UP" || echo "DOWN")
  echo "[$i] Ollama: $status"
  [ "$status" = "DOWN" ] && break
done
```

| # | Check | Pass? | Notes |
|---|---|---|---|
| 6a | `/stop` returns `{"ok": true, "action": "stop"}` immediately (no 20s freeze) | | |
| 6b | Ollama process goes DOWN within ~35 seconds | | |
| 6c | Ollama does NOT respawn (check again 30s later: `curl http://localhost:11434/api/tags`) | | |
| 6d | WebUI state badge shows `STOPPING` briefly then `STOPPED` | | |
| 6e | Provider endpoint dot turns red + "Offline" | | |
