## Test 6 — STOP button actually stops Ollama (svcmgr fix)

**What we're testing:** The STOP button sends `bootout` to launchd via the correct plist (`homebrew.mxcl.ollama.plist`), preventing KeepAlive respawn. This was broken in v0.1.

```bash
KEY=<api-key>

# Trigger STOP
curl -s -m 5 -X POST -H "X-API-Key: $KEY" http://localhost:8764/stop

# STOP returns immediately (non-blocking). Poll for Ollama going down.
for i in $(seq 1 15); do
  sleep 2
  status=$(curl -s --connect-timeout 2 http://localhost:11434/api/tags 2>/dev/null && echo "UP" || echo "DOWN")
  echo "[$i] Ollama: $status"
  [ "$status" = "DOWN" ] && break
done
```

| # | Check | Pass? | Notes |
|---|---|---|---|
| 6a | `/stop` returns `{"ok": true, "action": "stop"}` immediately (no 20s freeze) | PASS | Returned in ~2s (SSH overhead); response `{"ok":true,"action":"stop"}` |
| 6b | Ollama process goes DOWN within ~35 seconds | PASS | DOWN at first poll (3s) — launchctl bootout is fast |
| 6c | Ollama does NOT respawn (check again 30s later) | PASS | Still DOWN after 30s — KeepAlive no longer triggers after bootout |
| 6d | WebUI state badge shows `STOPPING` briefly then `STOPPED` | PASS | STOPPING badge captured in screenshot; transition fast (~2s) |
| 6e | Provider endpoint dot turns red + "Offline" | PASS | Confirmed in browser |
