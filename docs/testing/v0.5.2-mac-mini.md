## Test 3 ÔÇö Install without Ollama present

**What we're testing:** Running `install.sh` on a machine where Ollama is not installed. The installer should complete (llm-valet itself installs fine), but the service should start in a degraded state and surface a clear error ÔÇö not crash silently.

```bash
# Uninstall Ollama first
brew services stop ollama
brew uninstall ollama

# Re-run installer
curl -fsSL https://raw.githubusercontent.com/LegionForge/llm-valet/main/install/install.sh | bash
```

**Finding ÔÇö Python removed as Ollama dependency:**
`brew install ollama` installs `python@3.14` as a dependency. When Ollama is uninstalled via `brew uninstall ollama`, Homebrew autoremoves Python as an unneeded formula ÔÇö leaving the machine with no Python 3.11+ on PATH. This is standard Homebrew dependency management behavior, not a bug.

**Implication for users:** Anyone on macOS who only had Python because Ollama installed it will hit this if they ever uninstall Ollama before llm-valet. Re-installing Ollama also brings Python back as a side effect. This should be noted in the README Prerequisites ÔÇö Python is a separate install requirement from Ollama on macOS.

Installer failed at step 1 with a clear, actionable error:

```
Error: Python 3.11 or newer is required.
  macOS: brew install python
  Ubuntu/Debian: sudo apt install python3
  Download: https://python.org
```

This is correct behavior ÔÇö fail fast with guidance rather than crash silently.

**Resolution for continued testing:** `brew install python`.

## Test 4 ÔÇö Startup overcommit warning

**What we're testing:** When a model is loaded whose memory footprint exceeds `ram_pause_pct`, llm-valet logs a warning at startup and surfaces it in the WebUI.

Re-install Ollama, pull a model, set `ram_pause_pct` to a value below the model's footprint, restart the service.

```bash
brew install ollama && brew services start ollama
ollama pull qwen3.5:0.8b

# Set ram_pause_pct artificially low to trigger the warning
# Edit ~/.llm-valet/config.yaml: ram_pause_pct: 10

# Load the model
curl -s -X POST http://localhost:8765/load \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen3.5:0.8b"}'

# Restart the service to trigger startup check
launchctl kickstart -k gui/$(id -u)/com.legionforge.llm-valet
sleep 5
curl -s http://localhost:8765/status | python3 -m json.tool | grep overcommit
```

|---|---|---|---|
| 4a | Warning appears in `valet.log` at startup | Ôťů | `WARNING overcommit at startup ÔÇö loaded model exceeds RAM pause threshold; watchdog auto-pause will not trigger` |
| 4b | Warning includes model name, actual MB/%, and threshold | Ôťů | `model=qwen3.5:0.8b model_mb=2013 model_pct=12.3 ram_pause_pct=10` |
